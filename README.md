# 📊 Analyzing E-Learning Platform Purchases using MySQL

## 📌 Project Overview

This project analyzes an **E-Learning Platform** database using **MySQL** to uncover valuable business insights from learner purchases, course performance, and revenue trends.

The database consists of learner information, course details, and purchase transactions. By writing SQL queries, the project identifies customer purchasing behavior, top-performing courses, revenue-generating categories, and business opportunities for improving learner engagement.

---

## 🎯 Objectives

* Design a relational database for an online learning platform.
* Analyze learner purchasing behavior.
* Identify the highest spending learners.
* Discover the most popular courses.
* Evaluate revenue generated by different course categories.
* Detect courses with no purchases.
* Generate actionable business insights using SQL.

---

## 🛠️ Technologies Used

* **MySQL**
* SQL (DDL, DML & Analytical Queries)

---

## 🗂️ Database Structure

The project contains three main tables:

* **Learners**
* **Courses**
* **Purchases**

These tables are connected using primary and foreign key relationships to perform relational analysis.

## 🔍 Key Insights

* High-value learners purchased multiple courses, contributing significantly to total revenue.
* Data Analytics and Programming courses received the highest number of purchases.
* AI and Data Analytics generated the highest revenue among all categories.
* Several learners enrolled in courses across multiple categories.
* SQL queries successfully identified courses with zero purchases, helping evaluate underperforming offerings.
